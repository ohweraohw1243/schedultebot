# Мой личный бот-календарь 🗓️

Это мой пет-проект, который я написал, чтобы собрать всё свое расписание в одном удобном месте. Я учусь в универе в НГТУ и параллельно подрабатываю репетитором. Из-за этого расписание всегда лежит в двух разных местах: у вуза на сайте (статичное), а у учеников в CRM-системе "МойКласс" (постоянно меняется, бывают отмены и переносы за пару часов).

Поэтому я решил сделать бота, который сам лезет по всем моим сайтам, собирает расписание, сохраняет его к себе в базу и выдает мне удобную ссылку для Apple Calendar (и виджетов типа Structured). А если кто-то из учеников внезапно перенес занятие — бот напишет мне об этом в телегу.

## Что внутри (Технологии)
- **Python 3.11+** как основной язык.
- **BeautifulSoup4** — чтобы парсить HTML табличку расписания универа.
- **Requests** — чтобы тянуть скрытый JSON из "МойКласс".
- **SQLite + SQLAlchemy** — сохраняю все занятия в локальную базу данных, чтобы не качать их каждый раз и находить изменения.
- **aiogram 3** — сам телеграм-бот, который дает кнопки управления и присылает уведомления.
- **APScheduler** — фоновая задача, которая проверяет расписание каждые 30 минут.
- **FastAPI + iCalendar** — маленький сервер, который на лету генерирует ссылку `.ics`, чтобы можно было добавить расписание в приложение на телефоне.
- **Docker / docker-compose** — упаковал всё это в контейнеры, чтобы потом удобно закинуть на сервер 24/7.

## Как это запустить у себя
1. Клонируем репозиторий и ставим библиотеки:
```bash
pip install -r requirements.txt
```
2. Создаем файл `.env` и вписываем туда токен от телеграм-бота:
```env
BOT_TOKEN=твой_токен_от_ботфазера
```
3. Инициализируем локальную базу данных SQLite:
```bash
python -m database.setup
```
4. Запускаем сервер календаря и самого бота:
```bash
uvicorn api:app --host 0.0.0.0 --port 8000
python bot.py
```
*(Либо можно просто запустить `docker-compose up -d --build`, если есть докер)*
